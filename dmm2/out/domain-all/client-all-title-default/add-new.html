In Oxygen XML Editor, how do I add a new document to Kuali KMS?

Adding a new document to Kuali KMS using Oxygen

You must have configured an Oxygen XML Editor project. For details, see How do I connect Oxygen XML Editor to Kuali KMS?
  1. In Oxygen XML Editor, select File > New... .
  2. Under the From templates tab, choose the appropriate DITA template.
  3. Edit the document.
  4. Select File > Save to URL...
  5. Make sure the connection is configured using the same settings as initially configuring WebDAV.
  6. Next to File URL:, change the default name Untitled.xml to a new filename with the file extension dita. Leave the rest of the path unchanged.
    Note: If you are importing an existing KB document, use the docid as the name of the file; e.g., ddud.dita.
  7. Click OK.
    Note: After saving once to the KMS, when you choose File > Save , your changes will be saved to the remote repository. You do not have to choose Save to URL... again.

Now that you have saved the new file to the server, you need to add the new file to the Oxygen project file to make the new file appear in the Oxygen Project view.

  1. Above the main editing view, click the tab for the new file to select it.
  2. In the Project view, right-click the default folder and, from the context menu, select Add Edited File.
  3. Verify that the new file appears in the default folder.
  4. Edit the id attribute of the topic element and change it to the docid of the KB document you are importing with -dmm2 appended; e.g., ddud-dmm2
The new file is now saved on the server and added to the Oxygen project file.
